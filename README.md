# Движок браузерной MMO-игры (Django)

> **Ранний прототип** — не является готовым продуктом.  
> Репозиторий временно публичный для демонстрации наставнику. После получения фидбэка будет переведён в **приватный режим**.


Это **настраиваемый движок** для браузерных MMO на Django.  
Вся игровая сущность — **локации, монстры, предметы, дроп, торговцы** — управляется **через Django Admin**, без изменения кода.

Основная идея: **разделение логики и контента**. Разработчик пишет механики, а контент-менеджер наполняет игру через админку.

---

## Текущее состояние (демонстрационная версия):

- **Система перемещения** между локациями.
- **Базовый бой**: функция возвращает победу игрока (заглушка).
- **Расчёт шанса дропа** с монстра и **начисление** в инвентарь.
- **Торговец** в городе (покупка/продажа предметов).
- **Одна локация**, 4 монстра, но **драться можно только с одним**.
- Шаблоны и изображения загружены **для демонстрации** (будут исключены из репозитория в будущем).

**Шаблоны и изображения — временные, используются только для визуализации текущего состояния.**

---

## Планы:

1. **Наполнить игру контентом** до базового играбельного уровня:
   - Добавить ресурсы и систему их добычи.
   - Расширить боёвку (реальные расчёты).
   - Добавить больше локаций, монстров, предметов.
2. **Реализовать коммуникацию**:
   - Почта между игроками.
3. **Экономика**:
   - Аукцион.
4. Только после этого — **новые механики** (квесты, гильдии и т.д.).
